# Exercise 2 — Command vs. Plain Prompt

## Try this once as a plain prompt

> Run `npm test`, filter failures, and tell me which TodoMVC spec category each failure falls into.

Then author `/todomvc-spec-check` (Phase 6 Track A) and run it on the same repo.

## Compare

- Which was faster to type?
- Which produced more consistent output across two runs?
- Which would you share with teammates?

## Break-point heuristic

Write the plain prompt from scratch five times — once a day, for a week. If you wrote it the same way each time, it was a good candidate for a command. If each rewrite was different, it probably shouldn't be promoted.

## Debrief

What signal tells you a plain prompt is ready to be promoted to a command? (Hint: it's not just "I typed it twice".)
